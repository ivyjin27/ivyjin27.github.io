数据分析方法论总结

发表于 2019-02-28 | 分类于 数据分析

一眨眼，19年已经过去1/6了。学习真是件逆水行舟，不进则退的事情~ 既然没学习先对过去的经验总结下。

1.数据分析的意义
最近想到数据其实是用户和APP的对话，他们用自己的行为告诉APP喜欢什么、不喜欢什么，有什么痛点……
数据分析师，需要通过这些纷杂的“聊天记录”，窥探业务本质，诊断业务存在的问题，探索出机遇。

2.如何去了解一个新的业务
不同产品都具有一些共性，仅以目前的经验，认为应当是下面几个方面：

a.规模现状
这部分遵循AARRR模型，就是了解当前阶段的新增、活跃、留存、变现和自分享情况。
同时还包括产品的新增和活跃趋势，判断产品处于什么样的生命周期：是新生期、成长期、成熟期还是衰退期？ 以及在历史进程中，主要的增长点和原因。

b.产品形态
产品形态是从产品功能层面，看用户的转化和流失。
用一句人话说就是，用户从哪儿来，都干了什么，最后去了哪里，在哪些地方出现流失，原因是什么。到原因部分，可能会需要多维度的观察。

c.用户行为
用户在这个业务干了什么事情，是不是我们期望的。也就是用数据回答这几个问题：
用户都从哪里来？如果是站外新增，那都是从哪些渠道来的，我们都投入过什么？如果是站内转化，那是从哪些业务来的，这些业务直接有什么共性，是什么原因链接了这两个业务。

d.用户特性
也就是用户画像，譬如年龄、性别、城市、行为习惯……

3.如何找到优化方向
通过上面对现在业务的洞察后，应该是会有一些发现的：
产品漏斗：目前到这个业务的主要流失点在哪里；
用户行为：譬如用户都是从哪些地方转过来来的，如何去做高度的捆绑
用户画像：业务触达的人群主要是哪些，怎么去吸引同样特质，但是还没有转化成客户的用户； 未触达的用户为什么没有来，是什么原因，怎么优化或者创造场景。

数据指标

发表于 2019-01-13 | 分类于 数据分析

前言：在经历大半年数据运营的工作后，回头看《精益数据分析》，感觉受益颇丰，思维又得到了提升。

数据的价值

“人类提供灵感，机器负责验证”。数据不能够创造业务，但能为业务保驾护航：
事前，验证是否存在“灵感”的先验条件
事中，监测运营中业务的效果和存在的问题、找出可优化点
事后，在特定阶段进行评估。
这也是我目前所体会到的，数据化运营和数据产品的价值：掌握数据，可以更好的验证灵感。

什么是好的数据指标

前提

与商业模式紧密挂钩：营收来源、支出成本、客户数量和客户获取策略的效果

准则

具有比较性，简单易懂，是一个比率，能够针对指标的变化改变行为。
寻找指标需走查：
a.定性指标和量化指标
b.虚荣指标和可付诸行动指标
c.探索性指标与报告性指标

问题类型 隐藏信息
我们知道我们知道的 是可能并不为真的事实，须经数据检验
我们知道我们不知道的 是可通过调研解答的问题，可使之成为我们的行为基准，并流程化
我们不知道我们知道的 是直觉，需要评估并训练以提高其效能和效率
我们不知道我们不知道的 是探索，蕴含着我们自身独特的优势，能带来有趣的顿悟

史蒂夫.布兰克说：创业公司应该把时间花在探索并得出一个可拓展、可复制的商业模式上

d.先见性指标和后见性指标
e.相关性指标和因果性指标

测试方法

市场细分、同期群分析、A/B测试和多变量分析

市场细分

细分市场是一群拥有某种共同特征的人，例如手机APP市场中的客户端是来自安卓，还是IOS等

同期群分析

比较相似群体随时间的变化，例如知识付费领域经常关注不同阶段新增用户的ARPU值

A/B测试和多变量测试

在同一段时间段对不同被测试群体提供不同的体验。
其中A/B测试是每一时间段仅改变某一方面，衡量其效果和影响；多变量测试是同时改动多个方面，看哪个与结果相关性最大。

数据分析框架

AARRR

戴夫.麦克卢尔的海盗指标说，也是目前互联网应用最为普遍的一套框架，描述了用户/客户/访客须经历的5个环节：
获取用户（Acquisition）、提高活跃度（Activation）、提高留存率（Retention）、获取营收（Revenue）和自传播（Referral）

创业增长金字塔

肖恩.埃利斯的创业金字塔着眼于创业公司在找到产品与市场契合点之后，该如何增长。简单分为三个阶段：
产品与市场相契合（决定把什么卖给谁） -> 厘清思路（找到可防御的高壁垒的竞争优势，并进一步巩固） -> 规模增长

长漏斗

这属于一种常用的分析方法了，在产品优化中经常会用到，体现了从获取用户 -> 用户发生期待行为的一个转化路径。

第一关键指标

OMTM：One Metric That Matters，第一关键指标，指的是在当前阶段，需要集中全部注意力的数字。
使用第一关键指标的理由，与第一关键指标的特征有：
它回答了现阶段最重要的问题
它促使得出初始基线并建立清晰的目标
关注整个公司层面的健康
它鼓励一种实验文化
例如：餐饮业的前一天人工成本占收入的比例

数据分析误区

假设数据没有噪声、忘记归一化、排除异常点、包括异常点、忽略季节性、抛开技术谈增长、数据呕吐、谎报军情的直播、“不是在这儿收集的”综合征（多尝试结合不同来源的数据进行分析）

写在最后：数据在已知的产品迭代方面，能够检验效果、很快的发现问题并找到机会，可以把已有的业务做到极致。但是所有颠覆性的改变，都是源于人类的直觉、或者说灵感。即：”渐进式的改变可以达到局部极限，创新则可能导致全局洗牌”。

资源投放效率评估模型

发表于 2018-12-23 | 分类于 python

2018年9月接了专辑完播预测模型，2018年12月底才算基本完成第一期。不知道最后投入应用效果会如何，但对我自己的意义是很大的。了解到很多统计学知识，包括时间序列、最小二乘回归、泊松回归、负二项回归、零膨胀Poisson回归等，还初步掌握statsmodels的应用。其中收获最多的是：a.首次重度依赖python完成一个项目，已经可以熟练进行数据读取、处理、建模和可视化呈现，b.将实际问题转化为数据模型，以及更好的数据解读能力。

模型概览：

目的：提高内容分发效率
简介：通过可能影响专辑完播的因素，包括：曝光、历史完播、上传节目数和是否工作日来拟合专辑的当前完播。其中曝光的系数是曝光吸收度，历史完播系数是专辑自成长系数，上传节目数是主播投入，是否工作日是周期性影响因素。

项目历程

2018.8:大部分时间在整理数据，一开始计划直接清洗日志数据，结果发现非常麻烦麻烦麻烦…… a.历史数据中脏数据太多，数据存储格式又有较大差异；b.数据量太大，等待时间长
2018.9.10:尝试时间序列，发现只用历史专辑完播得到拟合效果已经很好，只用少数时间点存在较大误差

2018.9.18:用前7日完播、专辑质量分、是否工作日、上传节目数建模等变量进行建模；并对量级大的数据做对数处理
2018.10.10:确定主要变量为前7日完播、专辑质量分、是否工作日、上传节目数；并尝试用零膨胀模型消除源数据中零样本数据的干扰
2018.10.26:考虑到数据为count data，尝试用poisson回归进行拟合
2018.11.08:最终确定用OLS回归，前期工作暂且结束，准备输出数据产品
2018.11.22:完成数据输出，开始结果分析
2018.12.3:模型输出系数波动较为剧烈，推测是变量之间的共线性影响
2018.12.14:进行变量删减，单用曝光、t-1完播、上传声音数和是否工作日后，各变量的系数波动明显变缓
2018.12.21:确定先投入应用，监测监测效果

一些值得记录的tips

1.尽量少在整理数据上用太多时间，分析师主要还是验证思路可行性，理顺业务和数据的逻辑
2.应用新算法时，先看document里面的简单案例，copy代码执行一遍；再套用对应业务数据，以及了解其中的统计学知识
3.在数据应用的时候，准确性很重要，但是要普遍应用，往往需要更重视简单性和可解释性
4.最终结果呈现的时候，通过一些典型的、或者大家都知道的案例，会比展示一堆分布图有效
5.在展示分布的时候，如果散点图拥挤，可以用四分位数作折线图进行展示

时间序列

发表于 2018-11-29 | 分类于 算法

曾经一个项目里面有用到时间序列，但是当时由于赶进度，主要精力放在了模型实现上。但是“知其然而不知其所以然”的知识点，对我来说是最容易遗忘的，所以在周末学习一下概念部分~

时间序列算法

常见时序模型

模型名称 描述
平滑法 常用于趋势分析和预测，根据平滑技术可区分为移动平均法和指数平滑法
趋势拟合法 把时间作为自变量，相应的序列观察值作为因变量，简历回归模型。可分为线性拟合和曲线拟合
AR模型 xt = φ0 + φ1xt-1 + φ2xt-2 + … + + φpxt-p + 𝜀t ，是以前p期的序列模型为自变量、随机变量Xt为因变量建立线性回归模型
MA模型 xt = μ + 𝜀t - θ1𝜀t-1 - θ2𝜀t-2 - … - - θq𝜀t-q，随机变量Xt的趋势xt与以前各期的序列值无关，建立xt与前q期的随机扰动𝜀t-1,𝜀t-2,…,𝜀t-q的线性回归模型
ARMA模型 xt = φ0 + φ1xt-1 + φ2xt-2 + … + + φpxt-p + 𝜀t - θ1𝜀t-1 - θ2𝜀t-2 - … - - θq𝜀t-q,随机变量Xt和前期p期的序列值和前q期的随机扰动都有关
ARIMA 对于非平稳时间序列，如果差分后显示出平稳序列的性质吗，则称为差分平稳序列，可通过ARIMA模型进行拟合

AR模型

自相关系数ACF具有拖尾性
偏自相关系数PACF具有p阶截尾性

MA模型

自相关系数ACF具有q阶截尾性
偏自相关系数PACF具有拖尾性
