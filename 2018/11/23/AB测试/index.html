AB测试

发表于 2018-11-23 | 分类于 数据分析

AB测试貌似是互联网业内产品和运营的法宝，当方案遭到质疑的时候，只要拍着胸脯说我会先做个AB测试，根据数据效果上线。但是实际情况下，ABtest大多会流于形式。
从业以来，参与过不少产品的ABtest过程，但是最后的分析通常比较简单，大多情况下都是对比两种情况下的一个点击，或者转化率的差异。所以在工作中对于ABtest的掌握也不太深刻，先整理下再网上拔到的资料，如果工作中有更多体会，就再补充吧。

AB测试的基础知识包括3部分：What(AB测试是什么),When(什么时候应用AB测试),How(如何运行AB测试)

AB测试的概念

概念

AB测试是为Web或App界面或流程制作两个（A/B）或多个（A/B/n）版本，在同一时间维度，分别让组成成分相同（相似）的访客群组随机的访问这些版本，收集各群组的用户体验数据和业务数据，最后分析评估出最好版本正式采用。

根据上述概念可知，AB测试的核心有3步：流量分隔，拆分为对照组和实验组 -> 指标采集 -> 效果分析

优点

简单易用，结果可靠
充分考虑用户行为，优化改版功能后再全量覆盖
降低风险，降低调研时间

缺点

不同用户之间存在体验差异
线上需要同时运行两套逻辑代码，维护成本高
泄露产品部分功能

什么时候运用

AB测试是可以考虑广泛应用的，但是在下面这几种场景的需求会比较急迫：
产品优化依靠经验主义，不能保证新的产品版本一定会有业绩提升
重大产品功能很难决策，不确定那个方案效果最优
如果改版失败，业绩损失无法挽回

如何运行

在AB测试之前，先进行AA测试，保证样本特征一致，设定转化率浮动的标准，确定样本大小
容易出现的问题：
对照组系统偏差 -> 先进行AA测试
样本数不够 -> Power Analysis
实验过早结束
网络效应带来偏差 -> 排除网络效应
